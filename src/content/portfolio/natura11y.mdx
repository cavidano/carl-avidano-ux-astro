## Why I built Natura11y

Natura11y is an accessibility-first design ecosystem I designed and built to keep design decisions, component behavior, and production code aligned. As my passion for accessibility grew, so did my desire to become a stronger front-end developer and understand how design systems work from conception through production.

What began as a personal learning project has grown into an open-source ecosystem containing shared CSS and JavaScript foundations, Figma libraries, an icon system, React components, Astro documentation, Storybook, and coordinated package releases.

The project represents the full design-system lifecycle: defining principles, creating design and code foundations, engineering accessible interactions, documenting component behavior, distributing packages, and applying the system to production websites.

</CaseStudyOverview>


<TextBlock>

## Ecosystem architecture

As Natura11y grew, I needed an architecture that could support the expanding ecosystem without allowing its different parts to drift apart. I brought everything into a monorepo and organized it around the single responsibility principle: Core owns the shared styles and vanilla JavaScript, Icons manages the source artwork, icon data, and generated font, and React builds accessible components with Core's styles and utilities. The Astro documentation and Storybook live alongside those packages, giving me one place to develop, test, and document the system.

This structure allows me to make a decision once and carry it through the entire ecosystem. The same icon data powers the documentation and Storybook, React reuses Core instead of duplicating its foundations, and each package exports only what other parts of the system need. The result is a maintainable architecture whose parts can evolve and release independently while continuing to work together as one design system.
